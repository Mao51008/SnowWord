# SnowWord 操作指南

这份文档收敛常用命令，方便日常调试、发消息和服务器运维。

## 1. 常用 npm 命令

### 登录并添加账号

```bash
npm run login
```

用途：
- 扫码登录 iLink Bot
- 把新账号写入数据库

注意：
- 建议先停掉正式服务再登录新账号
- 登录完成后再重启 `pm2`

### 查看账号列表

```bash
npm run accounts
```

输出里会看到类似：

```text
1. [启用] xxx (bot_xxx) -> user_xxx@im.wechat
2. [启用] yyy (bot_yyy) -> user_yyy@im.wechat
```

说明：
- 前面的 `1`、`2` 是命令里可用的序号
- 括号里的 `bot_xxx` 是更稳定的 `account_id`
- 实际运维更推荐直接用 `account_id`

### 查看账号状态

```bash
npm run state -- 1
```

或：

```bash
npm run state -- bot_xxx
```

用途：
- 查看当前人格、关系、情绪、主动状态
- 查看剩余订阅天数和到期时间

### 设置账号剩余订阅天数

```bash
npm run subscription:set -- 1 30
```

或：

```bash
npm run subscription:set -- bot_xxx 30
```

说明：
- 会把该账号的订阅到期时间设置为“从现在起 30 天后”
- 到期前 1 天，系统会自动给对应用户发提醒
- 设置后会重置“已提醒”状态

### 在当前基础上续费增加天数

```bash
npm run subscription:add -- 1 30
```

或：

```bash
npm run subscription:add -- bot_xxx 30
```

说明：
- 会在当前到期时间基础上继续往后加天数
- 如果当前订阅已经过期，就从现在开始重新累计
- 续费后会重置“已提醒”状态

### 只查看订阅信息

```bash
npm run subscription:show -- 1
```

或：

```bash
npm run subscription:show -- bot_xxx
```

会显示：
- 剩余订阅天数
- 到期时间
- 上次提醒时间

### 删除账号

```bash
npm run account:remove -- 1
```

或：

```bash
npm run account:remove -- bot_xxx
```

## 2. 电脑端本地调试

这个模式不走 iLink，直接在电脑终端里和 agent 对话，适合调试 prompt、语气和人设。

### 启动本地调试会话

```bash
npm run chat -- 1
```

或：

```bash
npm run chat -- bot_xxx
```

进入后直接输入内容即可：

```text
You> 老板好烦啊
小雪> 啊，发生什么了？
```

退出：

```text
/exit
```

说明：
- 会复用该账号的人设、长期记忆和数据库里的近期上下文
- 不走 iLink
- 不会真的向微信发消息

## 3. 后台主动发消息

### 给账号绑定的默认用户发消息

```bash
npm run send -- 1 "晚点记得吃饭，别又拖太晚。"
```

或：

```bash
npm run send -- bot_xxx "晚点记得吃饭，别太晚。"
```

### 给指定用户发消息

```bash
npm run send -- 1 some_user@im.wechat "这事听起来真的挺烦的。"
```

说明：
- 这条消息会通过 agent 账号发出去
- 默认按单条文本发送
- 会写入当前账号的消息记录

## 4. 微信内可用 Slash 指令

这些命令是在微信里直接发给 agent 的。

### 切换人格

```text
/切换人格 小雪
/切换人格 初雪
```

等价英文：

```text
/persona xiaoxue
/persona chuxue
```

### 清除长期记忆

```text
/清除记忆
```

等价英文：

```text
/clear-memory
```

### 清除历史对话

```text
/清除历史对话
```

等价英文：

```text
/clear-history
```

### 清除日程

```text
/清除日程
```

等价英文：

```text
/clear-schedule
```

### 清除全部数据

```text
/清除所有
```

等价英文：

```text
/clear-all
```

说明：
- 会清掉记忆、历史对话、日程、状态
- 账号绑定信息会保留

### 自定义人格补充

```text
/自定义人格提示词 你说话要更轻一点，更像熟悉后的恋人型陪伴
```

等价英文：

```text
/persona-prompt 你说话要更轻一点，更像熟悉后的恋人型陪伴
```

清空自定义补充：

```text
/自定义人格提示词
```

## 5. 本地开发命令

```bash

```

用途：
- 本地开发调试

注意：
- 线上不要长期用这个

### 构建

```bash
npm run build
```

### 类型检查

```bash
npm run typecheck
```

## 6. 服务器常用命令

### 查看 PM2 状态

```bash
pm2 status
```

### 查看日志

```bash
pm2 logs snowword --lines 100
```

### 重启服务

```bash
pm2 restart snowword
```

### 停止服务

```bash
pm2 stop snowword
```

### 保存开机启动配置

```bash
pm2 save
```

## 7. 更新发布流程

推荐顺序：

```bash
cd /opt/SnowWord
cp store/messages.db store/messages.db.bak-$(date +%F-%H%M%S)
git pull --rebase
npm ci
npm run build
pm2 restart snowword
pm2 logs snowword --lines 100
```

说明：
- 先备份数据库
- 再拉代码、安装依赖、构建、重启
- 重启后马上看日志

## 8. 数据位置

常见文件和目录：

- `store/messages.db`
- `data/`
- `logs/`
- `.env`

说明：
- 这些文件都很重要
- 升级前建议先备份

## 9. 建议

- 运维时优先使用 `account_id`，不要长期依赖序号
- 新增账号前，建议先 `pm2 stop snowword`
- 线上运行用 `pm2 + npm start`
- 本地调试语气和 prompt 时，优先用 `npm run chat`
